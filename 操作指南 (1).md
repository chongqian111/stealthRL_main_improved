## 二、代码文件说明

### `train_local.py`（训练）

**CFG 配置字典说明**：

```python
CFG = {
    # ── 模型路径 ──────────────────────────────────────────
    "base_model":   "/data/.../Qwen2.5-0.5B-Instruct",
    "output_dir":   "checkpoints/stealthrl-qwen2.5-0.5b",

    # ── 数据 ──────────────────────────────────────────────
    "num_samples":  5000,   # 训练总样本数（按顺序取，不随机）
                            # 测试用500，正式用5000~10000
    "token_min":    100,    # 论文标准 Appendix F
    "token_max":    500,

    # ── LoRA ──────────────────────────────────────────────
    "lora_rank":    16,     # 0.5B 模型用 16 够了
    "lora_alpha":   32,

    # ── 训练超参 ──────────────────────────────────────────
    "learning_rate":   2.8e-4,  # 论文值，不随步数变化
    "batch_size":      8,       
    "num_generations": 4,       # GRPO group size，测试用4，正式用8
    "max_steps":       100,     # 测试用100，正式用2000
    "kl_coef":         0.05,    # 论文 KL 惩罚系数

    # ── 奖励权重（论文 Section 3.2）───────────────────────
    "alpha":          1.0,   # 检测逃逸权重
    "beta":           0.1,   # 语义相似度权重
    "roberta_weight": 0.6,   # RoBERTa 在集成中的权重
    "fdgpt_weight":   0.4,   # Fast-DetectGPT 在集成中的权重
}
```

**奖励公式（对齐论文）**：
```
pens = 0.6 × P_RoBERTa + 0.4 × P_FastDetectGPT
R    = 1.0 × (1 - pens) + 0.1 × E5语义相似度
```

**截断修复**：prompt 中包含词数提示：
```
"Paraphrase the following text while preserving its meaning.
Write approximately {N} words.

Original: {原文}

Paraphrase:"
```

**⚠️ 重要：改了数据量必须删旧文件**
```bash
# 只要修改了 num_samples 或 token_min/max，必须先删：
rm -f data/processed/train.jsonl data/processed/eval.jsonl \
      data/processed/esl_validation.jsonl data/processed/native_validation.jsonl
```
原因：脚本检测到文件存在会直接跳过数据准备，用旧数据。

### `evaluate.py`（评估）

**四个检测器（对齐论文 Section 4.2）**：

| 检测器 | 训练时 | 说明 |
|---|---|---|
| RoBERTa | ✅ 训练集成（权重0.6） | 测直接效果 |
| Fast-DetectGPT | ✅ 训练集成（权重0.4） | 测直接效果 |
| Binoculars | ❌ held-out | 测跨架构迁移 |
| MAGE detector | ❌ held-out | 测跨架构迁移 |

**两个数据集**：MAGE test split + UCAS val.csv

**输出指标**：AUROC、TPR@1%FPR、ASR@1%FPR、TPR@5%FPR、E5相似度

- 输出：`submission.csv`


```bash

```

批量评估找最优 checkpoint：
```bash
for ckpt in 500 1000 1500 2000; do
    sed -i "s|checkpoint-[0-9]*\"|checkpoint-${ckpt}\"|" evaluate.py
    echo "=== checkpoint-${ckpt} ===" >> results/compare.txt
    python evaluate.py 2>&1 | grep -E 'AUROC|ASR' >> results/compare.txt
done
cat results/compare.txt
```

## 五、常见问题

**Q: 改了 num_samples 但数据条数没变？**
```bash
# 必须删旧文件再重跑
rm -f data/processed/*.jsonl
python train_local.py
```

**Q: OOM 显存不足？**
```python
# train_local.py 里减小：
"batch_size":      4,   # 从 8 降到 4
"num_generations": 4,   # 从 8 降到 4
```

**Q: 训练中断怎么恢复？**
```bash
# 直接重跑，已配置 resume_from_checkpoint=True
python train_local.py
```

**Q: 训练和评估同时跑 OOM？**
```bash
# 必须先停训练
kill $(pgrep -f train_local.py)
python evaluate.py
# 评估完再恢复训练
python train_local.py
```

**Q: Fast-DetectGPT 加载报错 embed_dim 问题？**
```python
# 确保使用 AutoModelForCausalLM 而非 GPT2LMHeadModel
fdgpt_model = AutoModelForCausalLM.from_pretrained(fdgpt_name, ...)
```

## 六、评估指标解读

| 指标 | 含义 | 目标 |
|---|---|---|
| AUROC | 检测器区分AI/人类能力，0.5=随机猜 | 越低越好 |
| TPR@1%FPR | 严格阈值下的检测率（主要指标） | 越低越好 |
| ASR@1%FPR | 攻击成功率 = 1 - TPR | 越高越好 |
| TPR@5%FPR | 宽松阈值下检测率（补充） | 越低越好 |
| E5相似度 | 改写前后语义保留程度 | >0.85 |
